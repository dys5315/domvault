# The Graph Compounds Like Interest

> A note that just sits there adds *principal*. A note that connects to many others earns on the
> *running total*. The value of a knowledge graph is the edge set, not the node count — and like
> compound interest, it goes slow-then-sudden.

## The connection
Simple interest grows on principal alone — `A = P(1 + r·t)`, linear. Compound interest grows on
the running total — `A = P(1 + r)^t`, exponential, because each period's gain earns the next. A
knowledge base compounds the same way. A new isolated note adds linear principal (one more node).
A new **synthesis note that links many prior notes** grows the accumulated connectivity — value
on the running total, not the principal.

## Why it matters
The defensible asset in a second brain is the **curated edge set**, not the raw note count.
Anyone can copy your pipeline in an afternoon; nobody can copy months of connections. The
reinvestment rate `r` is your **synthesis cadence**. Skipping a synthesis cycle isn't a flat
loss — it's forgone compounding that permanently lowers the curve.

## So what
- Treat the periodic synthesis pass as a non-skippable reinvestment payment.
- Measure the system by **edge growth / connectivity**, not note count.
- Prune aggressively — dead links and stale half-ideas are *negative* interest on the principal.
