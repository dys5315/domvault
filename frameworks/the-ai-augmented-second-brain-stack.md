# The AI-Augmented Second Brain Stack

> A second brain has three layers: a markdown-and-graph memory, AI agents as workers, and a loop
> where the system improves itself. The differentiator isn't the model — it's edge quality.

## The connection
A cluster of ideas about running an AI-powered knowledge system all point at the same setup: a
local, plain-text vault as the memory, AI agents as the workers, and a loop where the system
improves itself. Stitched together, they form one blueprint.

## The stack
- **Memory layer.** Markdown notes + a link graph. Plain text you own, that any tool can read, with
  the graph as the structure.
- **Worker layer.** Coding agents and sub-agents that read, write, ingest, and synthesize against
  the vault — the "AI worker that runs while you sleep" pattern.
- **Shared-context layer.** A shared brain across agents so multiple workers operate on the same
  memory without drifting, plus reusable skills/plugins that encode repeatable workflows.
- **Self-improvement loop.** The vault generates its own candidate connections for a human to
  review — exploration writes edges, review keeps them honest.

## Why it matters
The point isn't reading *about* this stack — it's operating it. The defensible asset isn't the model
(anyone can call one); it's the curated, linked memory and the loop that keeps enriching it. The
stack is portable across any domain you work in.

## So what
- Keep the memory in plain text + links so it outlives any single tool.
- Formalize the repeatable workflows (ingest, synthesize, publish) as skills/commands rather than
  one-off prompts.
- Treat the self-improvement loop as the engine: it's what turns a notebook into a brain.
