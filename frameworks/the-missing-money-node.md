# The Missing Money Node (Idea→Revenue Audit)

> A feature is only done when its conversion edge works end-to-end — no dead CTAs, no placeholder
> stubs, no gate that approves work but releases nothing. Every feature needs a working money node.

## The pattern
The common failure: every stage of the funnel gets built except the one where money changes hands.
The product implements `Data → Work → Insight → Decisions` and stops one arrow short of
`→ Revenue`. The symptoms are recognizable:
- The top-of-funnel CTA is a stub — a button that pops an alert or a `mailto:` instead of capturing
  the lead and carrying it into the system built to receive it.
- Approval gates exist and are audited, but **nothing is bound to them** — the gate proves a
  milestone was accepted and then does nothing with the one thing acceptance was supposed to
  trigger (the release of funds, the invoice, the next tranche).
- The cheapest artifact has the most complete payment path while the flagship has none —
  monetization gravity inverted, the toy can take a dollar but the business cannot.

## Why it matters
A trust machine that grades and audits work is one step from completion: the gates *grade* the work,
so payment should be the **payout function of the grade**. If the revenue edge isn't wired, the
audit log is a seam waiting for the one edge that makes the whole promise real. Two structural
failures recur portfolio-wide: no first-dollar path wired into the flagship, and no venture with
time-bound kill criteria.

## So what
1. **Bind money to gates.** On gate-approved → generate the phase invoice / payment link for that
   tranche. The audit log is already the event seam.
2. **Revive dead CTAs.** Every conversion button posts a real lead into the system, with a real
   follow-up, using infrastructure that already exists.
3. **Put pricing inside the product.** Generate the price from the captured inputs; kill "as quoted
   separately."
4. **Keep a first-dollar scoreboard.** One visible number: $ collected, deals signed, leads captured
   this week. Goals can't be hit unmeasured.
5. **Run a kill-criteria pass.** Add time-bound, measurable kill criteria to each active project
   *before* calling anything "on track."
