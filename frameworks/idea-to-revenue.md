# Idea to Revenue (A Validation Gate)

> Before building, make an idea pass a validation gate — and define how you'll kill it before you
> start.

## The validation gate
An idea earns a build only when it clears three checks:
- **Problem clarity** — is the problem quantified and specific, not a vague "this would be nice"?
- **Audience specificity** — can you name the exact person who has this problem, in behavioral
  terms, not a broad demographic?
- **First-dollar path** — is there a concrete, wired route to the first paid transaction? Not "we'll
  monetize later" — the actual edge where money changes hands.

## Kill criteria
Decide how the idea *dies* before you start:
- **Define before starting** — kill criteria written after launch are rationalizations, not
  thresholds.
- **Time-bound** — "if X hasn't happened by date Y."
- **Measurable** — a number you can check, not a feeling.

## So what
- Run every idea through the gate before committing build effort; an idea that can't name its
  first-dollar path or its kill criteria isn't ready.
- Revisit the kill criteria on schedule — the discipline only works if you actually honor the
  threshold you set.
